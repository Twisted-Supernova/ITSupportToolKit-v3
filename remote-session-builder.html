PSExec Quick Reference

  • -s - Run as SYSTEM account
  • -i - Interactive mode (opens GUI on remote system)
  • -h - Run with elevated token (if available)
  • -u - Specify username
  • -p - Specify password (will prompt if omitted)
  • -c - Copy executable to remote system
  • -f - Force copy even if file exists
  • -d - Don't wait for process to terminate
Security Note: PSExec requires SMB access and admin rights on the remote system. It may be flagged by antivirus. Download from Microsoft Sysinternals: https://learn.microsoft.com/sysinternals/downloads/psexec

PowerShell Remoting Setup

  • Enable on remote PC: Run Enable-PSRemoting -Force as admin
  • Trust hosts (workgroup): Set-Item WSMan:\localhost\Client\TrustedHosts -Value "*"
  • Test connection: Test-WSMan -ComputerName PC-NAME
  • Default Port: HTTP (5985), HTTPS (5986)
  • Firewall: Ensure WinRM service is allowed through firewall
CredSSP Warning: CredSSP authentication allows credential delegation but increases security risk. Only use in trusted environments.

RDP Command Line Options

  • /v: Specify computer to connect to
  • /admin - Connect to console session (admin session)
  • /f - Start in full screen mode
  • /w: Screen width
  • /h: Screen height
  • /public - Runs RDP in public mode (no caching)
  • /span - Match RDP width/height with virtual desktop
  • /multimon - Use multiple monitors
  • /restrictedAdmin - Connect in Restricted Admin mode
Restricted Admin Mode: Prevents credentials from being sent to remote computer. Use for enhanced security, but some features may not work. Requires Windows Server 2012 R2 or Windows 8.1 or later.
